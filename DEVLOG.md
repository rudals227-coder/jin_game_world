# 개발 이력 (DEVLOG)

Jin Game World — 여러 미니게임을 모아놓은 허브형 웹앱.
PC에서 개발하고, 아이패드 Safari에서 URL 접속으로 플레이한다.

> 아키텍처/게임 추가법 등 "현재 구조" 설명은 [CLAUDE.md](CLAUDE.md) 참고.
> 이 문서는 "무엇을 왜 만들었는지"의 시간순 기록이다.

---

## 2026-07-11

### 1. 프로젝트 초기 구성
- **스택 결정**: 바닐라 JS + HTML5 Canvas, 빌드는 Vite. (프레임워크 없이 미니게임을 독립 모듈로)
- **플레이 방식 결정**: 아이패드 Safari에서 **URL 접속**. 백엔드 없는 순수 클라이언트 앱.
  - 오프라인/PWA는 범위에서 제외 → 온라인 정적 호스팅(GitHub Pages 등)으로 배포하기로.
- **허브 구조**:
  - 해시 라우팅(`#/` 허브, `#/game/<id>` 게임) — 서버 설정 없이 정적 URL로 동작.
  - **게임 모듈 계약**: 각 게임은 `mount(container) → unmount()` 를 export. 라우터가 화면 전환 시 정리 담당.
  - 공용 엔진 유틸: `engine/canvas.js`(DPR 스케일+리사이즈), `engine/input.js`(Pointer Events 통합), `engine/loop.js`(RAF 루프).
  - 게임 목록은 `games/registry.js` 한 곳에서 관리 → 허브 카드 자동 생성.
- iPad 규약: 뷰포트 줌 차단, `touch-action:none`, `overscroll-behavior:none`, safe-area 여백.

### 2. 첫 게임 — 쿤판(클로츠키/화용도) 슬라이딩 퍼즐
- **모델/뷰 분리**: `puzzle.js`(이동 규칙·승리 판정), `editor.js`(배치·검증), `levels.js`(기본 배치+localStorage 저장) 는 캔버스를 모르는 순수 로직. `index.js`가 렌더+입력 담당.
- **두 가지 모드**:
  - 풀기 모드: 드래그로 블록 슬라이드, 큰 말을 하단 출구로 → 클리어.
  - 직접 배치 모드: 팔레트에서 피스 선택 → 탭으로 배치/제거 → 겹침·경계 검증 → "이 문제 풀기"/"저장".
- 순수 모델 단위 테스트 21개 통과, 빌드/개발 서버 동작 확인.

### 3. 쿤판 그래픽 개선
- 나무 프레임 + 움푹한 판 + 그라데이션/광택/베벨/그림자로 입체감.
- 출구 표시(맥동 점선+화살표), 클리어 오버레이 애니메이션.
- **피드백 반영**: 처음엔 말 위에 한자(曹操/將 등)를 넣었으나, 사용자 요청으로 **글자 제거** → 깔끔한 색 블록으로.

### 4. 슬라이드 애니메이션
- 각 말에 "표시 위치"(float)를 두고 매 프레임 논리 위치로 easing 수렴 → 칸 단위 점프 대신 부드러운 미끄러짐.
- 프레임레이트 독립(`1 - e^(-22·dt)`).

### 5. 메인 화면 스팀 스타일 리디자인
- **피드백 반영**: "검은색만 있어서 심심하다, 스팀처럼 만들어달라."
- 추천 배너(히어로) + 커버 아트 카드 그리드로 재구성. 배경에 은은한 컬러 글로우.
- 커버 아트는 **외부 이미지 없이 SVG로 생성**(오프라인/정적 배포 안전). 쿤판은 실제 보드 모양 미리보기.
- registry에 `tagline/tags/accent/cover/featured` 메타 추가 → 카드·태그·커버 자동 생성.

### 6. 두 번째 게임 — 벽돌깨기(Breakout)
- 실시간(물리) 게임 예제. RAF 루프로 매 프레임 `step(dt)`(공/패들/벽돌 충돌) 후 렌더.
- 조작: 드래그로 패들 이동, 탭으로 발사/재시작. 상태: `ready/playing/won/lost`.
- 점수·목숨 HUD, 무지개 벽돌, 회전/리사이즈 시 위치 비율 스케일로 상태 유지.
- registry 등록 + SVG 커버 추가. 메인 화면에 카드 2개 확인.

### 7. 효과음 추가
- **피드백 반영**: "소리도 나게 해달라."
- `engine/audio.js` — Web Audio API로 효과음 실시간 합성(음원 파일 0개).
- 음소거 토글(🔊/🔇, localStorage 저장), iOS 첫 터치에서 오디오 잠금 해제.
- 벽돌깨기(벽/패들/벽돌/놓침/클리어), 쿤판(슬라이드/배치/클리어)에 연결.

### 8. 세 번째 게임 — 달 착륙선(Lunar Lander)
- 중력에 맞서 엔진 분사로 착륙장에 저속·수평 착륙하는 아케이드 게임. 연료 제한.
- 물리: 중력/분사(기체 방향)/회전/연료. 지형 생성 + 평평한 착륙장(깃발). 착륙 판정(착륙장 위 + 저속 + 수평이면 성공, 아니면 추락).
- 조작: 화면 하단 버튼 3개(◀🔥▶, 터치), PC는 방향키/스페이스. 탭으로 시작/재시작.
- 사운드: `engine/audio.js`에 **지속 분사음**(필터링 노이즈 루프) `startThrust/stopThrust` 추가. 착륙=성공음, 추락=실패음. 음소거 시 분사음도 정지.
- HUD(연료 바/속도), 우주 배경+별, 회전/리사이즈 시 위치·지형 비율 스케일. registry 등록 + SVG 커버.

### 9. 로컬 실행 바로가기
- **목표**: 로컬에서 개발 서버를 더블클릭으로 켜기 (깃헙 배포는 나중에).
- `start-dev.bat` 추가 — 의존성 자동 설치(최초 1회) → `npm run dev -- --host --open` 실행. PC 브라우저 자동 열기 + 아이패드용 Network 주소 안내.
- 바탕화면에 `게임월드 개발서버.lnk` 바로가기 생성(이 PC 전용, 커밋 대상 아님).
- **트러블슈팅**: 처음엔 `.bat`을 UTF-8로 저장 → cmd(cp949)에서 한글이 깨지고 깨진 글자가 명령으로 오해석되어 오류. → **배치 메시지를 순수 ASCII(영문)로 교체**해 해결(비ASCII 0바이트 확인).

### 10. 확대(줌) 방지
- **증상**: 게임 중(특히 달 착륙선 분사 버튼 반복/동시 탭) 화면이 확대됨.
- **원인**: 브라우저 기본 제스처(더블탭·핀치 확대). iOS Safari 는 `user-scalable=no` 를 무시하기도 함 → 의도한 기능이 아니라 버그.
- **해결**:
  - `main.js`: Safari 제스처 이벤트(`gesturestart/change/end`)와 더블탭(`touchend`/`dblclick`), 멀티터치 `touchmove` preventDefault.
  - `styles.css`: `.game-screen` 전체에 `touch-action: none` → 게임 화면 안 더블탭/핀치 완전 차단(허브 스크롤은 유지).

### 11. 깃허브 배포 (GitHub Pages)
- **목표**: 로컬 서버 없이 URL 접속으로 플레이.
- `.github/workflows/deploy.yml` 추가 — main 브랜치 push 시 자동 빌드 후 GitHub Pages 배포(actions: checkout/setup-node/upload-pages-artifact/deploy-pages @v4, Node 20).
- git 저장소 초기화 + 첫 커밋(24개 파일). `base: './'` + 해시 라우팅 덕분에 서브경로(`/저장소명/`)에서도 정상 동작.
- 저장소: `github.com/rudals227-coder/jin_game_world` → 배포 주소 **https://rudals227-coder.github.io/jin_game_world/** (배포 성공 확인).
- **트러블슈팅**:
  - push 거부(`fetch first`): 원격을 README와 함께 생성해 이력 불일치 → `git push --force`(빈 원격이라 안전)로 해결.
  - Actions의 "Node.js 20 deprecated" 경고 2건 → 깃허브 측 런타임 안내일 뿐, 오류 아님(무시).
- **이후 배포**: `git add . && git commit -m "..." && git push` 하면 자동 재배포.

### 12. 벽돌깨기 아이템 시스템
- 벽돌을 깨면 확률(약 22%)로 아이템 캡슐 낙하 → 패들로 받으면 효과 발동. 좋은:나쁜 ≈ 3:1 가중치.
- 6종(1차 세트): 🔥파이어볼(맞은 벽돌+상하좌우 폭발), ⚡멀티볼(공 분열, 최대 8개), ➤관통(안 튕기고 뚫기), ↔확장/><축소(패들 폭, 상쇄), 🐢슬로우(공 감속). 시간제 효과는 8~10초.
- **리팩터링**: 단일 공 → `S.balls[]` 배열(멀티볼). 모든 공 낙하 시 목숨 감소. 효과 타이머 `S.effects`, 낙하 아이템 `S.items`.
- HUD에 활성 효과 뱃지+남은시간 바. 파이어=주황 발광, 관통=보라 공.
- 검증: 임시 데모 장면으로 아이템/효과/멀티볼 렌더 스크린샷 확인 후 데모 코드 제거.

### 13. 네 번째 게임 — 광산 채굴 (팩맨 스타일)
- 팩맨 코어(미로 이동+수집+추적자 회피+파워 역공)를 동굴/채굴 테마로. **클리어=보석 전부 획득**, 조작=화면 D-pad(+방향키).
- **차별점: 벽 파기** — 갈색 파기벽(`%`)을 눌러 ~0.6초 뚫어 지름길/도주로 생성. 몬스터는 통과 불가.
- 모델/뷰 분리: `maze.js`(순수: 타일·ASCII 레벨 파싱·통행/파기/수집/보석 판정) + `index.js`(뷰: 격자 정렬+방향 버퍼링 이동, 몬스터 AI, 렌더).
- 몬스터 2종: 추적형(그리디 최단), 배회형(랜덤). 다이너마이트 먹으면 7초 겁먹음(파랑)→접촉 시 스폰 귀환+콤보. 워프 터널.
- 조작: `.dpad` 십자 방향패드(styles.css) + 키보드. 목숨 3, 잡히면 위치 리셋.
- 검증: `maze.js` 단위 테스트 24개 통과(보석 도달 가능성 포함), 빌드/렌더/플레이 스크린샷 확인.
- **v1 제외(향후)**: 어둠/랜턴, 산소 게이지, 함정, 추가 아이템, 다중 레벨, 몬스터 3종+.
